# 合格证打印系统 - 项目概览

## 🎯 项目简介

合格证打印系统是一个专业的车辆合格证打印管理系统，采用前后端分离架构，实现了从参数维护到打印执行的完整流程管理。

## 📊 系统架构

### 技术架构
```
┌─────────────────────────────────────────────────────────────┐
│                        用户界面层                            │
│                  React 19.2 + Ant Design 6.0                │
└──────────────────────┬──────────────────────────────────────┘
                       │ HTTP/REST API
┌──────────────────────┴──────────────────────────────────────┐
│                      应用服务层                              │
│                   Spring Boot + JWT                         │
├──────────────────────────────────────────────────────────────┤
│                      业务逻辑层                              │
│    参数管理 | 订单车 | 打印控制 | 补打 | 报表统计           │
└──────────────────────┬──────────────────────────────────────┘
                       │ MyBatis-Plus
┌──────────────────────┴──────────────────────────────────────┐
│                       数据层                                 │
│              MySQL 8.0 + Druid连接池                        │
└─────────────────────────────────────────────────────────────┘
```

## 🔑 核心业务流程

### 1. 总体打印流程
```
车辆入库 → 系统校检 → 判断重打/补打
                 ↓
          判断二类底盘/订单车
                 ↓
          加载参数 → 打印预览
                 ↓
          执行打印 → 记录日志
```

### 2. 二类底盘校验规则
- VIN前8位 + VSN前2位 **同时在维护表** → 二类底盘
- 仅一项匹配 → 报错
- 均不匹配 → 整车

### 3. 数据关系
- 参数表 ← 中间表 → 订单车信息（多对一）
- 打印日志 → 参数表（外键）
- 打印日志 → 操作员（外键）
- 补打记录 → 打印日志（外键）
